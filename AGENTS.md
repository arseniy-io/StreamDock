# Правила работы с проектом

Этот файл задаёт правила для разработчиков и ИИ-агентов, которые меняют проект.

## Назначение

Приложение локально анализирует ссылки YouTube, скачивает видео или аудио и создаёт транскрибацию без облачных API. Основная платформа - Windows, основной сценарий запуска - `start.bat`, адрес приложения - только `127.0.0.1`.

## Язык и подача

- Пользовательский интерфейс, README и понятные сообщения об ошибках пишутся на русском языке.
- Технические имена классов, функций и API пишутся на английском.
- Сначала объясняется пользовательский смысл, затем технические детали.
- Не показывать пользователю traceback, команды `yt-dlp` или внутренние аргументы FFmpeg.

## Архитектура

- Не складывать backend-логику в `app/main.py`.
- HTTP-маршруты находятся в `app/api`, бизнес-логика - в `app/services`, схемы - в `app/models`.
- Долгие операции выполняются как фоновые задачи и не блокируют HTTP-запрос.
- Не добавлять базу данных, React, Electron, Docker и внешние API без отдельного решения.
- Пути строить через `pathlib.Path`, а управляемые пользователем имена очищать перед записью.

## Безопасность

- Слушать только `127.0.0.1`.
- Разрешать только URL с `http` и `https` и поддерживаемых хостов.
- Не использовать `shell=True` и не собирать команды строковой конкатенацией.
- Не принимать от frontend произвольные аргументы для `yt-dlp`, FFmpeg или Whisper.
- Не реализовывать обход DRM, платных ограничений и чужой авторизации.
- Полный текст исключений сохранять в локальном логе, пользователю показывать безопасное объяснение.

## Правила интерфейса

- Главный источник компонентов и структуры - [shadcn/ui](https://ui.shadcn.com/).
- Визуальные идеи и отдельные блоки можно искать в [21st.dev](https://21st.dev/).
- [Aceternity UI](https://ui.aceternity.com/) и [Magic UI](https://magicui.design/) использовать только для редких микроэффектов, skeleton-состояний и hover-реакций.
- [Rapira](https://rapira.net/) использовать как ориентир по спокойным отступам, мягким скруглениям и ощущению качественного продукта.
- Tailwind UI, Tremor, библиотеки графиков, Google Stitch и Figma Community подключать только под конкретную задачу, которой нет решения в текущей системе.
- Источники служат ориентиром. Не копировать их React-код и не добавлять frontend-зависимости ради одного компонента.
- Дизайн-система и состояния подробно описаны в `DESIGN.md`. Любая новая страница должна ей соответствовать.

### Обязательное правило для новых элементов

- Перед добавлением элемента сначала найти ближайший существующий компонент в `app/static/index.html` и `app/static/styles.css` и переиспользовать его структуру, классы и токены.
- Нельзя оставлять нативный `select`, `input`, `checkbox`, `fieldset`, `button` или ссылку в стандартном оформлении браузера. Для нового контрола одновременно реализуются обычное, hover, `:focus-visible`, disabled, ошибочное и мобильное состояния, если они применимы.
- Не добавлять одноразовые цвета, отступы, скругления и тени, если нужное значение уже есть в токенах `:root` или правилах `DESIGN.md`.
- Если существующего паттерна нет, сначала описать новый компонент и его состояния в `DESIGN.md`, затем реализовать его в HTML/CSS/JS.
- HTML, CSS и JavaScript нового элемента изменяются и проверяются как единый комплект. Нельзя выпускать новую разметку без соответствующих стилей и поведения.
- После изменения интерфейса проверить, что браузер получил актуальные CSS и JavaScript, а не старые файлы из кеша.
- Новый элемент считается готовым только после визуальной проверки на ширине 1366, 1024 и 390 пикселей, проверки длинного текста и отсутствия горизонтальной прокрутки.

## Проверка изменений

- После backend-изменений запускать `.venv\Scripts\python.exe -m pytest`.
- После изменений импортов запускать `.venv\Scripts\python.exe -m compileall -q app tests`.
- После изменений интерфейса проверять минимум ширины 1366, 1024 и 390 пикселей.
- Проверять состояния: пусто, загрузка, успех, понятная ошибка, длинное название видео.
- Не переходить к следующему этапу, пока основной сценарий текущего этапа не работает.

## Работа с файлами

- Не откатывать существующие пользовательские изменения без прямой просьбы.
- Не записывать тестовые медиафайлы в репозиторий.
- Загруженные файлы хранятся в `downloads`, модели - в `models`, логи - в `logs`.
- Все текстовые файлы сохранять в UTF-8.

# Руководство пользователя StreamDock

Это руководство описывает обычную работу с приложением. Установка вынесена в [INSTALLATION.md](INSTALLATION.md), выбор моделей - в [MODELS.md](MODELS.md), а частые проблемы - в [TROUBLESHOOTING.md](TROUBLESHOOTING.md).

## Перед началом

1. Один раз выполните `install.bat`.
2. Запускайте программу через `start.bat`.
3. Работайте со страницей `http://127.0.0.1:8765`.

Не открывайте `app/static/index.html` двойным щелчком. Это только файл интерфейса, без локального сервера скачивание и транскрибация не работают.

Все результаты сохраняются в папке `downloads` внутри StreamDock. Модели находятся в `models`, технический журнал - в `logs/app.log`.

## Как добавить материал

### Одна ссылка

1. Вставьте публичную ссылку на видео или аудио.
2. Нажмите «Проверить ссылку».
3. Дождитесь названия, автора, длительности и доступных вариантов.
4. Выберите вкладку «Видео», «Аудио» или «Текст».

Поддерживаются YouTube, Rutube, VK Video, Vimeo, Dailymotion, TikTok, Twitch и SoundCloud. Фактическая доступность зависит от самого сайта, региона и состояния материала.

### Файл с компьютера

Нажмите «Транскрибировать файл с компьютера» и выберите видео или аудио до 20 ГБ. Локальный файл используется только для транскрибации и не отправляется во внешние сервисы.

Поддерживаются распространённые форматы, включая MP4, MKV, WebM, MOV, MP3, M4A, WAV, FLAC и Opus. Для чтения некоторых контейнеров нужен FFmpeg.

### Несколько ссылок или плейлист

1. Нажмите «Очередь» в правой части блока ссылки.
2. Вставьте до 50 ссылок, по одной на строку, или одну ссылку на плейлист.
3. Нажмите «Показать список».
4. Отметьте нужные элементы.
5. Выберите, что создать: транскрибацию, видео MP4 или аудио.
6. Нажмите «Запустить очередь».

Элементы обрабатываются последовательно. Для транскрибации очередь использует текущие настройки вкладки «Текст».

Если до очереди вы не открывали отдельный материал и не меняли настройки, используются значения по умолчанию: MP4 до 1080p, MP3 192 кбит/с или совместная русская транскрибация в Markdown. Чтобы применить другие настройки к очереди, сначала проверьте одну ссылку, настройте нужную вкладку, затем откройте очередь. Это ограничение текущей beta-версии.

## Скачать видео

1. Откройте вкладку «Видео».
2. Выберите одно из реально доступных разрешений.
3. Нажмите кнопку скачивания.

По умолчанию выбирается лучшее доступное качество до 1080p. Если изображение и звук на сайте находятся в разных потоках, StreamDock скачивает их отдельно и объединяет через FFmpeg в MP4.

Примерный размер показывается только тогда, когда источник сообщает достаточно данных. Итоговый файл может немного отличаться по размеру.

## Скачать только аудио

Откройте вкладку «Аудио» и выберите формат:

| Формат | Когда использовать |
| --- | --- |
| **MP3** | Максимальная совместимость с плеерами и телефонами |
| **M4A** | Хорошее качество без лишней перекодировки, когда источник уже отдаёт совместимый поток |
| **Оригинальный поток** | Сохранение исходного аудиоформата без обязательной конвертации |

Для MP3 доступны 128, 192, 256 и 320 кбит/с. Значение 192 кбит/с подходит большинству разговорных видео. Увеличение качества делает файл больше, но не улучшает уже плохой исходный звук.

## Обработать только фрагмент

После проверки ссылки откройте блок «Выбрать фрагмент» под превью. Начало и конец можно задать ползунками или временем в формате `ЧЧ:ММ:СС`.

Выбранный диапазон применяется к следующему скачиванию или транскрибации. Проверьте границы перед запуском операции.

## Создать транскрибацию

Откройте вкладку «Текст». Markdown уже выбран как основной формат.

### Источник текста

| Настройка | Что происходит |
| --- | --- |
| **Сначала готовые субтитры** | Приложение ищет ручные, затем автоматические субтитры и запускает модель только при их отсутствии |
| **Всегда распознавать аудио** | Звук обязательно обрабатывается выбранной локальной моделью |

Первый вариант работает быстрее. Второй нужен, если субтитры источника плохие или вы хотите проверить запись именно выбранной моделью.

### Режим распознавания

| Режим | Рекомендация |
| --- | --- |
| **Совместный** | Русские технические видео, вебинары и обучение. GigaAM распознаёт речь, Whisper `large-v3` проверяет сложные места |
| **GigaAM** | Быстрая транскрибация обычной русской речи на CPU |
| **Whisper** | Английский, смешанная речь и другие языки; можно выбрать размер модели |

Для русскоязычного технического видео с приоритетом качества выберите «Совместный», «Русский» и «Всегда распознавать аудио».

### Модели Whisper

- `tiny` и `base` - быстрый черновик;
- `small` - повседневный баланс;
- `medium` - более точный, но тяжёлый вариант;
- `large-v3` - максимальное качество Whisper с самой высокой нагрузкой.

Первая загрузка выбранной модели может занять время. Затем она хранится в `models` и повторно не скачивается.

### Форматы результата

- **MD** - читаемый Markdown с заголовком, сведениями об источнике и смысловыми блоками;
- **TXT** - обычный текст;
- **SRT** - субтитры для видеоплееров и монтажа;
- **VTT** - субтитры для браузера и веб-плееров.

Можно выбрать несколько форматов одновременно.

### Дополнительные настройки

- «Добавить временные метки» вставляет время перед смысловыми блоками;
- «Объединить реплики в абзацы» убирает стену коротких строк;
- «Убрать пустые короткие фрагменты» отбрасывает бессодержательные куски;
- «Пользовательские термины» помогает с фамилиями, названиями проектов и редкими словами;
- «Разделение по спикерам» добавляет подписи `Спикер 1`, `Спикер 2` и так далее.

Встроенный словарь терминов работает автоматически. Пользовательские термины вводите через запятую или с новой строки. Не добавляйте туда длинные предложения.

Если точное число участников известно, укажите его в настройках спикеров. Это обычно надёжнее автоматического определения, но увеличивает время и нагрузку.

## Готовые настройки для частых задач

| Задача | Рекомендуемые настройки |
| --- | --- |
| Быстро получить текст из YouTube | «Сначала готовые субтитры», MD |
| Русский технический вебинар | Совместный, русский, «Всегда распознавать аудио», MD и при необходимости SRT |
| Обычная русская лекция | GigaAM, русский, MD |
| Английское или смешанное видео | Whisper, автоопределение языка, `small` или `large-v3` |
| Подготовить субтитры | Временные метки, SRT или VTT |
| Интервью | Нужная модель, разделение по спикерам и известное число участников |

## Прогресс и отмена

Во время работы интерфейс показывает текущий этап, процент внутри этого этапа, скорость и оставшееся время, когда источник позволяет их определить.

Загрузка модели, анализ аудио и объединение через FFmpeg не всегда имеют точный процент. В таких этапах приложение показывает состояние процесса без выдуманной общей оценки.

Кнопка «Отменить операцию» останавливает задачу и связанные процессы. Уже готовые файлы предыдущих элементов очереди сохраняются.

## Результаты и редактор Markdown

После завершения появятся созданные файлы, их размеры и кнопка «Открыть папку». Исходный результат уже находится в `downloads`. Ссылка «Скачать» создаёт дополнительную копию через обычную папку загрузок браузера.

Файл с совпадающим названием не перезаписывается: StreamDock добавляет к новому имени номер. Текущие задачи и настройки интерфейса хранятся в памяти и после перезапуска приложения не восстанавливаются.

Для Markdown доступен компактный блок «Проверить Markdown»:

1. откройте блок;
2. найдите нужный фрагмент через поиск;
3. исправьте текст;
4. нажмите «Сохранить изменения».

Редактор не запускает модель повторно и не меняет другие форматы. Если вы исправили MD, соответствующие TXT, SRT и VTT автоматически не переписываются.

## Расширение Chrome

Расширение нужно для медиапотоков, которые появляются только внутри открытой страницы, включая доступные вам страницы личного кабинета. Оно не заменяет основное приложение и не запускает транскрибацию прямо из маленького окна.

После сохранения видео через расширение откройте StreamDock и выберите полученный файл для транскрибации. Полная установка и ограничения описаны в [руководстве расширения](../browser-extension/README.md).

## Где хранятся данные

- `downloads` - созданные видео, аудио и транскрибации;
- `models` - однажды загруженные локальные модели;
- `logs` - технический журнал;
- `.venv` - локальные Python-компоненты приложения.

При транскрибации своего файла приложению временно нужен дополнительный свободный объём для безопасной копии. Эта копия удаляется после успеха, ошибки или отмены.

StreamDock не создаёт аккаунт и не отправляет транскрибацию в облачный ИИ. Интернет всё равно нужен для получения медиа, установки компонентов и первой загрузки моделей.

## Обновление и помощь

Перед обновлением закройте активные задачи и StreamDock. Получите свежие файлы проекта, затем запустите `update.bat`.

Если что-то не работает, начните с [TROUBLESHOOTING.md](TROUBLESHOOTING.md). Вопросы можно задавать в GitHub Discussions, ошибки - оформлять в Issues. Никогда не публикуйте реальные cookies, приватные ссылки или полный журнал без проверки.

# Установка StreamDock

StreamDock предназначен для 64-разрядных Windows 10 и Windows 11. Основной пользовательский способ установки - `install.bat`, основной способ запуска - `start.bat`.

## Обычная установка

1. Скачайте архив релиза StreamDock и распакуйте его в постоянную папку или клонируйте репозиторий.
2. Запустите `install.bat` обычным двойным щелчком.
3. Дождитесь завершения проверки системных компонентов и установки Python-зависимостей.
4. Запустите `start.bat`.
5. Если браузер не открылся автоматически, перейдите на `http://127.0.0.1:8765`.

Не перемещайте папку проекта после установки расширения. Если путь изменился, повторно запустите `install.bat`, чтобы обновить регистрацию локального помощника Chrome.

## Что проверяет установщик

`install.bat` находит 64-разрядный Python 3.11+, подготавливает виртуальное окружение `.venv`, устанавливает зависимости из `requirements.txt` с проверенными ограничениями `constraints.txt`, создаёт служебные папки и пытается зарегистрировать локальный помощник расширения для текущего пользователя Windows.

Для полной работы нужны:

- Python 3.11 или новее;
- FFmpeg и FFprobe для объединения, конвертации и проверки медиа;
- Node.js 22 или новее для стабильной работы современного YouTube;
- доступ в интернет для установки зависимостей и первой загрузки моделей.

Python и Python-компоненты обязательны. Если FFmpeg или Node.js отсутствуют, установщик выводит предупреждение и команду установки, но не завершает настройку с ошибкой. После установки компонента запустите `install.bat` ещё раз, чтобы повторить проверку.

Если регистрация помощника Chrome не удалась, основное приложение всё равно можно запускать через `start.bat`. Расширение не сможет самостоятельно запускать и останавливать backend, пока помощник не будет установлен.

## Предварительная загрузка моделей

Запуск `download_models.bat` необязателен. Без него StreamDock сам скачает нужную модель при первом использовании соответствующего режима. Без дополнительных аргументов скрипт загружает рекомендуемый набор: GigaAM, детектор речи, Whisper `large-v3` и компактные модели разделения спикеров.

Предварительная загрузка полезна, если вы хотите:

- подготовить компьютер до начала длинной транскрибации;
- убедиться, что для модели хватает места;
- затем работать с локальными файлами без интернета.

Состав и ориентировочный объём моделей описаны в [MODELS.md](MODELS.md).

## Установка расширения Chrome

1. Завершите обычную установку через `install.bat`.
2. Откройте `chrome://extensions`.
3. Включите режим разработчика.
4. Нажмите «Загрузить распакованное расширение».
5. Выберите папку `browser-extension`.
6. Закрепите значок StreamDock на панели браузера.

Расширение использует локальный помощник, установленный только для текущего пользователя Windows. Системная служба и постоянно работающий фоновый сервер не создаются.

## Запуск и остановка

Запустите `start.bat`. Скрипт поднимает сервер только на `127.0.0.1` и открывает интерфейс в браузере. Если StreamDock уже работает, скрипт только открывает страницу и не создаёт второй сервер. При ручном запуске окно остаётся открытым; для остановки нажмите `Ctrl+C`.

Для ручной диагностики:

```powershell
.venv\Scripts\python.exe -m uvicorn app.main:app --host 127.0.0.1 --port 8765
```

Локальное приложение также можно запускать и останавливать из расширения Chrome. Управляемая остановка завершает созданные StreamDock процессы FFmpeg, FFprobe и другие дочерние процессы.

## Обновление

1. Завершите активные скачивания и транскрибации.
2. Закройте StreamDock.
3. Получите новые файлы релиза или обновите локальную копию репозитория.
4. Запустите `update.bat`, чтобы обновить зависимости текущей копии и переустановить локальный помощник.
5. После обновления снова запустите `start.bat`.

`update.bat` намеренно не скачивает исходный код и отказывается работать, пока запущен локальный сервер.

Загруженные модели и пользовательские результаты не должны скачиваться повторно без необходимости.

## Удаление

1. Завершите работу приложения.
2. Скопируйте нужные результаты из `downloads` в другую папку.
3. Запустите `uninstall.bat`, чтобы удалить регистрацию локального помощника Chrome.
4. Удалите распакованное расширение на странице `chrome://extensions`.
5. Удалите папку проекта.

Python, FFmpeg и Node.js, установленные отдельно от StreamDock, удаляются стандартными средствами Windows.

## Установка для разработки

```powershell
python -m venv .venv
.venv\Scripts\python.exe -m pip install --upgrade pip
.venv\Scripts\python.exe -m pip install -r requirements-dev.txt -c constraints.txt
.venv\Scripts\python.exe -m uvicorn app.main:app --host 127.0.0.1 --port 8765
```

Интерфейс будет доступен по адресу `http://127.0.0.1:8765`.

## Типичные проблемы

### Не найден Python

Установите 64-разрядный Python 3.11 или новее с официального сайта Python. Во время установки разрешите добавление Python в `PATH`, затем повторно запустите `install.bat`.

### Не найден FFmpeg

Установите FFmpeg вместе с FFprobe и убедитесь, что команда `ffmpeg -version` работает в новом окне PowerShell. После этого снова выполните `install.bat`.

### YouTube перестал анализироваться

Получите актуальные файлы StreamDock, затем запустите `update.bat`, чтобы установить проверенную версию загрузчика. Также убедитесь, что установлен Node.js 22 или новее.

### Порт 8765 занят

Закройте другую копию StreamDock или программу, использующую порт 8765. Не меняйте адрес на внешний сетевой интерфейс: приложение рассчитано на локальную работу через `127.0.0.1`.

### Модель не скачивается

Проверьте интернет, свободное место и локальный журнал `logs/app.log`. Повторный запуск не должен заново загружать уже сохранённые файлы модели.

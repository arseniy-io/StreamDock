# Решение проблем StreamDock

Начните с трёх безопасных действий:

1. закройте StreamDock и незавершённые задачи;
2. запустите `update.bat`;
3. снова запустите `start.bat` и повторите операцию.

Если проблема осталась, найдите подходящий раздел ниже. Технические подробности записываются в `logs/app.log`.

## Интерфейс открылся как огромная иконка или ничего не работает

Не открывайте `app/static/index.html` как обычный файл. Запустите `start.bat` и используйте только адрес:

```text
http://127.0.0.1:8765
```

Статус сервера и версию можно проверить по адресу `http://127.0.0.1:8765/api/health`.

## `install.bat` сообщает, что Python не найден

Нужен 64-разрядный Python 3.11 или новее. Установите его с [официальной страницы Python для Windows](https://www.python.org/downloads/windows/) и разрешите добавление Python в `PATH`.

После установки закройте старое окно терминала и снова запустите `install.bat`.

## Не найден FFmpeg или FFprobe

Без FFmpeg приложение не сможет объединять отдельные дорожки, конвертировать MP3 и обрабатывать часть локальных файлов.

Установка через Windows Package Manager:

```powershell
winget install --id Gyan.FFmpeg -e
```

После установки откройте новое окно PowerShell, проверьте `ffmpeg -version` и снова выполните `install.bat`.

## Не найден Node.js

Современный YouTube может требовать Node.js 22 или новее для выполнения служебных сценариев.

```powershell
winget install --id OpenJS.NodeJS.LTS -e
```

Затем откройте новое окно терминала и повторно запустите `install.bat`.

## Порт 8765 занят

Сначала проверьте, не запущен ли уже StreamDock. Если `http://127.0.0.1:8765` открывает приложение, вторая копия не нужна.

Если по этому адресу работает другая программа, закройте её обычным способом. Не завершайте неизвестные процессы только ради освобождения порта.

## Ссылка не принимается

StreamDock разрешает только `http` и `https` и поддерживаемые публичные источники. Проверьте:

- ссылка полностью скопирована из адресной строки;
- материал не удалён;
- ссылка не ведёт на локальный файл или внутренний адрес сети;
- источник перечислен в README;
- материал доступен в обычном браузере без регионального ограничения.

Для страницы личного кабинета используйте расширение Chrome, если видео воспроизводится в текущем сеансе и не защищено DRM.

## YouTube раньше работал, а теперь не анализируется

Сайты регулярно меняют способ выдачи видео. Получите свежую версию StreamDock и запустите `update.bat`. Также убедитесь, что Node.js установлен и доступен командой `node --version`.

Если ошибка повторяется только на одном видео, оно может быть приватным, возрастным, регионально ограниченным или удалённым.

## Нет нужного качества

Интерфейс показывает только варианты, реально найденные у источника. Если 1080p отсутствует, StreamDock не может создать его из более низкого качества без искусственного увеличения изображения.

Некоторые сайты отдают высокое разрешение отдельной видеодорожкой. Для объединения со звуком нужен FFmpeg.

## Не скачивается аудио

Проверьте FFmpeg и повторите операцию сначала с MP3 192 кбит/с. Если исходный поток несовместим с M4A, приложению потребуется конвертация.

«Оригинальный поток» сохраняет формат источника. Итоговый файл поэтому может иметь расширение WebM, Opus, OGG или AAC, а не MP3.

## Видео скачивается медленно

Скорость чаще всего ограничивается сайтом, интернет-соединением и выбранным качеством. Попробуйте:

- проверить другое видео с того же сайта;
- выбрать более низкое качество;
- временно отключить VPN или прокси, если они замедляют соединение;
- остановить другие большие загрузки;
- обновить StreamDock через `update.bat`.

Объединение дорожек после загрузки зависит уже от скорости диска и процессора, а не интернета.

## Первая транскрибация долго стоит на загрузке модели

Это нормально для крупной модели. Whisper `large-v3` занимает несколько гигабайт. Скорость зависит от интернета и диска.

Можно заранее подготовить рекомендуемый набор через `download_models.bat`. Повторный запуск проверяет кеш и не должен заново скачивать готовые файлы.

Если загрузка оборвалась, проверьте свободное место и запустите её снова. Не удаляйте папку `models` во время загрузки.

## Транскрибация идёт слишком долго

- Для обычной русской речи используйте GigaAM.
- Для русского технического видео используйте совместный режим.
- Полный Whisper `large-v3` выбирайте, когда его качество важнее времени.
- Отключите разделение по спикерам, если оно не нужно.
- Для проверки настройки сначала обработайте короткий фрагмент.
- Закройте игры, монтажные программы и другие тяжёлые приложения.

На компьютере без совместимой NVIDIA CUDA Whisper работает на CPU и может обрабатывать длинную запись заметно дольше её длительности.

## Прогресс остановился на одном проценте

Процент относится к текущему этапу, а не всегда ко всей задаче. Загрузка, объединение FFmpeg, подготовка модели и распознавание имеют разную скорость.

Если меняется текст этапа или работает диск/процессор, задача может продолжаться. При настоящей ошибке интерфейс покажет сообщение, а подробности появятся в `logs/app.log`.

## Не хватает оперативной или видеопамяти

Закройте тяжёлые программы, отключите разделение по спикерам и выберите меньшую модель Whisper. Для русского языка можно использовать GigaAM.

Если CUDA обнаружена, но памяти видеокарты недостаточно, приложение должно перейти на CPU. Обработка станет медленнее, но CUDA не является обязательной.

## Не открывается созданный файл

Нажмите «Открыть папку» и откройте файл из `downloads`. Убедитесь, что операция завершилась успешно и размер файла больше нуля.

Для редких контейнеров может понадобиться современный плеер. Если нужна максимальная совместимость, используйте MP4 для видео и MP3 для аудио.

## Расширение пишет, что локальный помощник не найден

1. Снова запустите `install.bat`.
2. Откройте `chrome://extensions`.
3. Найдите StreamDock и нажмите кнопку перезагрузки на его карточке.
4. Снова откройте расширение.

Если папка проекта была перемещена, `install.bat` нужно выполнить уже в новом расположении.

## Расширение не видит поток

1. Перезагрузите страницу с видео.
2. Запустите воспроизведение на несколько секунд.
3. Откройте StreamDock и нажмите кнопку обновления.
4. Проверьте, что видео действительно воспроизводится в Chrome.

Если поток по-прежнему не найден, страница может использовать DRM, нестандартный проигрыватель или временный адрес, который исчез до начала загрузки.

## В расширении нет кнопки «Остановить»

Перезагрузите распакованное расширение на странице `chrome://extensions` и снова выполните `install.bat`. Кнопка появляется только после подтверждённого соединения с локальным приложением.

Остановка отменяет активную загрузку и завершает процессы StreamDock, поэтому расширение запрашивает подтверждение.

## Где искать подробную ошибку

Откройте `logs/app.log` обычным текстовым редактором. Перед публикацией удалите из фрагмента:

- cookies и заголовки авторизации;
- приватные ссылки и параметры доступа;
- локальные пути с именем пользователя;
- названия и содержание личных материалов.

Для обычной ошибки создайте Issue по шаблону. Укажите версию StreamDock, источник без секретных параметров, выбранное действие, понятный текст ошибки и шаги воспроизведения.

Уязвимости и утечки секретных данных отправляйте по правилам [SECURITY.md](../SECURITY.md), а не в публичный Issue.

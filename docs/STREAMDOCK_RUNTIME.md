# StreamDock: расширение и локальная загрузка

## Как устроен основной сценарий

1. `browser-extension/background.js` замечает медиапотоки открытой вкладки и сохраняет кандидатов отдельно для каждой вкладки.
2. `browser-extension/popup.js` показывает лучший вариант первым. Полный `master.m3u8` имеет самый высокий приоритет.
3. После нажатия кнопки service worker создаёт постоянную вкладку `progress.html` и отправляет задачу в `POST /api/extension/download`.
4. `app/services/task_manager.py` запускает загрузку в фоновом потоке.
5. `app/services/extension_downloader.py` скачивает видео и звук, объединяет их в MP4 и перед сохранением проверяет обе дорожки через FFprobe.

Закрытие popup или вкладки прогресса не отменяет задачу. Загрузка продолжается в локальном приложении.
После успешного сохранения кнопка `Открыть папку` запускает Проводник Windows и выделяет конкретный скачанный файл, а не просто показывает каталог `downloads`.

## Отмена загрузки

Начиная с версии расширения `0.3.3`, кнопка `Отменить загрузку` выполняет полную двухфазную очистку:

1. Локальное приложение останавливает скачивание, удаляет частичный или уже успевший сохраниться файл и все временные HLS-фрагменты.
2. После подтверждения очистки расширение удаляет backend-задачу, свою запись загрузки и связанный с ней временный секрет.

Пока первый этап не завершён, интерфейс показывает состояние очистки. Повторное удаление уже очищенной задачи безопасно и считается успешным.
Отменой управляет фоновая часть расширения, поэтому вкладку прогресса можно закрыть сразу после нажатия кнопки. Незавершённая очистка сохраняется и автоматически возобновляется при следующем запуске фоновой части.

## Важная особенность Kinescope

Kinescope может отдать расширению дочерний `media.m3u8`, в котором есть только видео или только звук. Перед вызовом yt-dlp backend автоматически заменяет такой адрес на соседний `master.m3u8`, сохраняя параметры доступа после `?`. Master-плейлист содержит обе дорожки.

Записи кандидатов в service worker сериализованы отдельно для каждой вкладки. Это не даёт одновременным событиям master, video и audio перезаписывать друг друга.

## Запуск и остановка

- `install.bat` устанавливает локальный Native Messaging помощник.
- Кнопка `Запустить` в расширении поднимает приложение только на `127.0.0.1:8765`.
- Кнопка `Остановить` доступна и после ручного запуска: расширение получает временный секрет именно текущего процесса StreamDock и отправляет штатную команду backend.
- При запуске через расширение после штатной команды дополнительно закрываются все процессы внутри Windows Job Object.
- После изменения файлов распакованного расширения его нужно один раз перезагрузить на `chrome://extensions`.

## Где смотреть проблему

- `logs/app.log` - полная ошибка фоновой задачи.
- `logs/server-console.log` - запуск Uvicorn и HTTP-запросы.
- `logs/native-host.log` - запуск, остановка и ошибки локального помощника.
- `downloads` - только готовые файлы. Временные HLS-фрагменты создаются вне репозитория и удаляются после задачи.

## Быстрые проверки после изменений

```powershell
.venv\Scripts\python.exe -m pytest
.venv\Scripts\python.exe -m compileall -q app tests
node tests\extension_background_harness.js
```

Готовый MP4 обязательно проверять FFprobe: должны присутствовать хотя бы одна дорожка `video` и одна дорожка `audio`.

# Архитектура проекта

## Получение информации по ссылке

```text
Браузер
  -> POST /api/video/info
  -> app/api/video.py
  -> app/services/downloader.py
  -> yt-dlp: только чтение метаданных
  -> очищенная схема VideoInfoResponse
  -> интерфейс показывает видео и форматы
```

FastAPI также отдаёт статические файлы из `app/static`. Анализ `yt-dlp` выполняется в отдельном потоке, поэтому цикл обработки HTTP-запросов не блокируется.

## Ответственность каталогов

- `app/api` - проверка входных данных, HTTP-статусы и вызов сервисов.
- `app/services` - анализ, скачивание, конвертация, транскрибация и работа с файлами.
- `app/models` - схемы входных и выходных данных.
- `app/static` - локальный интерфейс без сборщика и внешних зависимостей.
- `downloads` - пользовательские результаты.
- `models` - модели Whisper, GigaAM, детектора речи и разделения спикеров.
- `logs` - технические журналы.
- `tests` - быстрые тесты без загрузки больших видео.

## Фоновые задачи

In-memory `TaskManager` уже создаёт задачу и сразу возвращает ID. Отдельный worker выполняет скачивание и FFmpeg, а frontend опрашивает статус раз в секунду. Отмена передаётся через `threading.Event` и останавливает `yt-dlp` без `shell=True`.

Фрагменты скачиваются во временную системную папку вне OneDrive. В `downloads` переносится только полностью собранный файл. Это защищает процесс от блокировки `.part` синхронизацией OneDrive и не оставляет временные фрагменты после отмены.

Сервис транскрибации поддерживает три локальных движка:

- `gigaam` - GigaAM v3 E2E RNNT распознаёт русскую речь через ONNX Runtime на CPU;
- `whisper` - faster-whisper распознаёт русский или английский и при наличии совместимой NVIDIA использует CUDA;
- `hybrid` - GigaAM обрабатывает запись целиком, затем эвристика выбирает только подозрительные технические фрагменты, а Whisper `large-v3` перепроверяет их.

В совместном режиме доля повторной проверки ограничена, а замена принимается только при безопасном совпадении границ и достаточной близости текста. Это не даёт короткому фрагменту Whisper удалить числа, начало или конец исходной реплики. Все движки возвращают единый список сегментов с таймкодами, который передаётся построителям MD, TXT, SRT и VTT.

Модели загружаются лениво и кешируются в памяти процесса. На CPU используются до 12 потоков; для Whisper применяются `int8`, пакетная обработка по четыре фрагмента и `beam_size=1`. Тяжёлые транскрибации выполняются последовательно, чтобы не конкурировать за процессор и память. Отмена проверяется между фрагментами GigaAM и перед каждым точечным запуском Whisper.

Транскрибация локального файла использует потоковую передачу тела запроса в управляемую временную папку. Frontend не передаёт произвольный путь с компьютера. API проверяет имя, расширение, лимит 20 ГБ и свободное место, после чего `TaskManager` запускает обычную фоновую задачу. Временная копия удаляется после успеха, ошибки или отмены.

`TaskRecord.progress` содержит процент только текущего этапа. Для скачивания дополнительно хранятся байты, скорость и оставшееся время, для распознавания - обработанная и полная длительность. Этапы с неизвестной длительностью возвращают `null`, и frontend показывает неопределённую полосу без фиктивного процента.

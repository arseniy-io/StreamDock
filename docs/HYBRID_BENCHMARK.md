# Проверка совместной транскрибации

Дата финального прогона: 18 июля 2026 года.

## Короткий вывод

Совместный режим рекомендуется по умолчанию для русских технических видео. На проверенных роликах он исправлял больше английских терминов, чем чистая GigaAM, и суммарно работал в 2,77 раза быстрее полного Whisper `large-v3`.

- GigaAM - когда важна максимальная скорость или в записи почти нет английских терминов.
- Совместный режим - основной выбор для русских лекций, вебинаров и технических видео.
- Whisper `large-v3` - когда допустимо ждать дольше ради максимально возможной точности.

Совместный режим не является новой обученной моделью. Это безопасная цепочка: GigaAM распознаёт запись целиком, приложение выбирает подозрительные фрагменты, Whisper проверяет только их, после чего принимаются лишь консервативные замены.

## Условия

- Intel Core Ultra 7 155H, 16 ядер и 22 логических потока;
- 31,5 ГБ оперативной памяти;
- Intel Arc без CUDA, поэтому весь тест выполнялся на CPU;
- приложение использовало до 12 потоков;
- Python 3.12.10, `faster-whisper` 1.2.1, `onnx-asr` 0.12.0 и `onnxruntime` 1.27.0;
- GigaAM v3 E2E RNNT INT8 и Whisper `large-v3` INT8;
- модели заранее находились на диске, каждый ролик запускался отдельной командой.

Внутри одного теста сначала запускалась GigaAM, затем совместный режим и полный Whisper. Время совместного режима включает GigaAM и точечную проверку. Полный Whisper мог повторно использовать уже загруженную в память модель, поэтому сравнение скорости не завышает преимущество совместного режима.

В качестве ориентира использовались ручные русские субтитры YouTube. Они не всегда дословно совпадают с речью, поэтому WER и CER показывают близость к субтитрам, а не абсолютную истинность. Чем ниже показатель, тем ближе текст к субтитрам.

## Финальные результаты

| Кейс | GigaAM: время / WER / CER | Совместный: время / WER / CER | Whisper: время / WER / CER | Проверено Whisper |
|---|---:|---:|---:|---:|
| Git | 23,9 с / 13,57% / 7,94% | 103,4 с / 12,80% / 7,62% | 256,5 с / 10,21% / 6,53% | 17,39% |
| SQL | 23,2 с / 15,52% / 9,81% | 90,5 с / 14,43% / 9,48% | 208,0 с / 13,42% / 9,39% | 15,09% |
| JSON | 16,3 с / 17,08% / 11,23% | 38,0 с / 16,95% / 10,67% | не запускался | 6,30% |
| DNS | 11,9 с / 13,29% / 5,92% | 34,3 с / 12,98% / 5,92% | 108,1 с / 11,13% / 6,92% | 9,90% |
| Обычная речь | 14,0 с / 2,34% / 0,62% | 14,0 с / 2,34% / 0,62% | 98,1 с / 2,11% / 0,78% | 0% |

На Git число найденных контрольных терминов выросло с 2 из 7 до 6 из 7, на SQL - с 2 из 8 до 4 из 8. На обычной речи подозрительных фрагментов не оказалось, поэтому Whisper не запускался и совместный режим занял ровно столько же времени, сколько GigaAM.

## Что улучшили после первой серии

- ограничили точечную проверку примерно 18% длительности записи;
- убрали запуск Whisper только из-за одного слабоуверенного фрагмента;
- запретили замену, если Whisper меняет числа или слишком сильно сокращает текст;
- сохранили начало и конец реплики при проверке фрагмента с запасом по времени;
- добавили словарь терминов из метаданных видео без придумывания текста;
- нашли ухудшение на JSON, ужесточили принятие замен и повторно получили улучшение;
- проверили, что обычная речь не вызывает ненужный запуск Whisper.

На этой выборке совместный режим ни разу не ухудшил GigaAM по WER. При этом полный Whisper всё ещё чаще даёт самый близкий к субтитрам текст, поэтому совместный режим следует воспринимать как удачный баланс скорости и качества, а не как гарантированную замену полного `large-v3` для любого аудио.

## Как повторить

Медиа и субтитры сохраняются только во временную папку Windows, а JSON - по указанному пути:

```powershell
.venv\Scripts\python.exe -m scripts.benchmark_hybrid git sql json dns ordinary --with-whisper --output "$env:TEMP\streamdock-hybrid-benchmark.json"
```

Первый запуск займёт дольше, если моделей ещё нет в локальном кеше.

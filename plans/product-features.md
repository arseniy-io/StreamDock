# Расширение функциональности StreamDock

Цель: усилить основной сценарий скачивания и транскрибации, не превращая главную страницу в перегруженную панель. Одна ссылка остаётся главным действием. Очередь, словарь, диапазон и редактор показываются как второстепенные компактные блоки. В расширение Chrome плейлисты и редактор не добавляются.

## Фаза 0. Общая основа

- [x] Зафиксировать схемы API и ограничения для новых параметров.
- [x] Сохранить совместимость существующих запросов frontend и расширения.
- [x] Добавить общие тестовые данные без загрузки больших медиафайлов.

## Фаза 1. Быстрая транскрибация из субтитров

- [x] Показывать наличие ручных и автоматических субтитров в данных видео.
- [x] Добавить режим «Автоматически»: ручные субтитры, затем автоматические, затем локальное распознавание.
- [x] Преобразовывать субтитры в MD, TXT, SRT и VTT с читаемыми блоками.
- [x] Сохранять источник и язык текста в Markdown.
- [x] Добавить прогресс, отмену, безопасный fallback и модульные тесты.

## Фаза 2. Встроенный и пользовательский словарь терминов

- [x] Добавить большой локальный словарь разработки, DevOps, облаков, безопасности и AI/ML.
- [x] Добавить компактное поле пользовательских терминов в настройках транскрибации.
- [x] Приоритизировать термины из названия, описания и пользовательского списка.
- [x] Передавать релевантные термины совместному режиму без выдумывания текста.
- [x] Ограничить размер и очистить пользовательский ввод, добавить тесты.

## Фаза 3. Второстепенная очередь и плейлисты

- [x] Дать выбрать элементы и действие: транскрибация, MP4 или MP3.
- [x] Выполнять элементы последовательно с общим прогрессом и отменой.
- [x] Показывать созданные файлы и ошибки отдельных элементов.
- [x] Не добавлять плейлисты в расширение Chrome.
